What is OOP?

    (Part 1)

OOP stands for Object Oriented Programming. There are many kinds of programming models. For example, Object Oriented, Procedural, Functional etc.

In OOP, we have Classes and Objects. A Class is a template for a set of Objects having same attributes or properties. For example, different Classes can be Humans, Animals, Mamals et.

We have other concepts in OOP like Abstraction, Inheritance, Polymorphism.

OOP was a result of programming design revolution that preceded procedural programming. Example of OOP languages are C++, C#, Java.

What is a Class?

A Class is a template for a Set of objects having common properties. For example, Customer is a Class. All Customers have certain set of properties like ID, Name, Type, all have business dealing etc.

For example,

	      
	      // Customer class
	      public class Customer
	      {
	      	public double ID {get; set;}
		public string Name {get; set;}
		
	      }

What is a Object?

Every instance of a Class is an Object. For example, if there is a Customer class, then, we can instantiate the Customer class and create an object.

C#

	      
	      // Customer class
	      public class Customer
	      {
	      	public double ID {get; set;}
		public string Name {get; set;}
		
	      }
	      
	      // Instantiating Customer to create an object
	      Customer cust = new Customer();
	      cust.ID = 1;
	      cust.Name = "Robert Curie";

What is an interace?

An interface is an Type in OOP, used to enforce certain behavior for Classes. For example, we may have different Classes like Manager, Lead and Trainee. Every class should have a Joining Date property. We can add Joining Date as a Property in an Interface. When a Class inherits this Interface, the Class must have a property called "JoiningDate".

C#

			// IEmployee interface
			interface IEmployee
			{
				DateTime JoiningDate {get; set;}
			}
			
			// Manager class
			public class Manager : IEmployee
			{
				DateTime JoiningDate {get; set;}
			}
			
			// Lead class
			public class Lead : IEmployee
			{
				DateTime JoiningDate {get; set;}
			}
			
			// Trainee class
			public class Trainee : IEmployee
			{
				DateTime JoiningDate {get; set;}
			}

What will happen, if Manager class does not have JoiningDate property? In this scenario, .net framework will throw an error, pointing out that Manager class must implement JoiningDate property.
